### Content-preservation contract (page ports)

When porting a page from legacy HTML to a React component:

1. **Do not alter, drop, or paraphrase any content.** A previous port
   attempt lost content. The constraint is load-bearing.
2. Copy text verbatim from the legacy HTML source. Do not retype.
3. Preserve every `<a href>` URL, every `<img>` `src` / `alt` /
   `loading` / `decoding` attribute, every `target` / `rel` attribute
   (even where `rel="noopener"` is missing — that's intentional legacy
   parity, not an oversight to fix in the port slice).
4. Preserve every inner-formatting tag: `<u>`, `<b>`, `<strong>`,
   `<span class="highlight-text">`, etc.
5. Preserve every class name the legacy `/assets/css/main.css` or the
   new `[data-page="..."]` rules in `app/globals.css` target.
6. Preserve the inline `style` on `<section class="hero">` — the
   `HiMMP-bg-*.jpg` substring is the parity-mask hook.
7. The route file (`app/<page>/page.tsx`) must continue to call
   `getLegacyPageContent(...)` and emit `<LegacyHeadExtras>`,
   `<LegacyStyles>`, `<LegacyScripts scripts={content.jsonLdScripts}>`
   and `<LegacyScripts scripts={content.bodyScripts}>` so the SEO
   surface (title, description, JSON-LD, meta) stays byte-identical to
   the legacy path.
8. Add the ported route to `REACT_OWNED_ROUTES` in the catch-all so
   `generateStaticParams` does not double-generate.

### Verification gates (run before requesting dual review)

Order matters — the cheap ones first:

```bash
cd nextjs-site
npm run typecheck            # cheapest, run first
npm run build                # cheap
npm run parity:content       # ~1s
npm run parity:text          # ~1s
npm run parity:visual        # 30–40s (run in background)
npx playwright test          # 50–60s (run in background)
```

`parity:text` strips tags and byte-compares the generated `<main>`
text against the legacy file. **It is the load-bearing safety net for
page ports.** Any added/removed/changed character in visible text
fails the gate.

`parity:content` enforces title, description, h1, and JSON-LD /
iframe / audio counts against `.migration/current-site-inventory.json`.

`parity:visual` only covers routes that are still in the migration-
preservation set (routes listed in `scripts/check-visual-parity.mjs`).
Pages that have been deliberately redesigned are removed from that
list and gain a Playwright body-smoke instead — see the about-body
slice (`tests/static-export.spec.ts:222`) as the reference pattern.

## Stack

- **Static**: pure HTML5 / CSS3 / ES6+ JavaScript — no build system, no
  package manager
- **Backend**: single PHP script (`contact-handler.php`)
- **Hosting**: any static web host; PHP optional

## Key Paths

- Pages: `index.html`, `about.html`, `approach.html`, `team.html`,
  `publications.html`, `videos.html`, `audio.html`, `contact.html`
- Main JS: `assets/js/main.js` — navigation, smooth scrolling,
  interactive components
- Audio player: `assets/js/audio-player.js` — comparison player for
  producer mixes
- Main CSS: `assets/css/main.css` — CSS custom properties in `:root`
- Responsive CSS: `assets/css/responsive.css`
- Assets: `assets/{audio,images}/` — images grouped by type
  (background / cover / logos / people / companies)
- Contact backend: `contact-handler.php`
- Submissions: `contact_submissions/`

## Conventions

- HTML5 with semantic elements and accessibility attributes
- BEM-like CSS class names; CSS custom properties throughout
- ES6+ JavaScript, event-driven, no external libraries
- MP3 audio for cross-browser compatibility
- Lazy loading + alt text for images
- Mobile-first responsive design (hamburger nav, flexible grids)
- Structured data + sitemap + meta tags for SEO
